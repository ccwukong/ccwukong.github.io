Deploy quickly to Vercel, or use AWS serverless for more control. Both options are supported and documented.

Vercel (recommended)

Fastest path to production. Zero DevOps for most stores.

  • Fork or upload the repo to GitHub
  • Connect the repo to Vercel and import the project
  • Set environment variables (Supabase URL, keys, OpenAI key)
  • Deploy — Vercel will build and publish automatically

AWS Serverless (Lambda + API Gateway)

For teams that need infra control and regional routing.

  • Package the Express server as a Lambda (serverless framework / AWS SAM)
  • Deploy API to API Gateway and configure a static frontend (S3 + CloudFront)
  • Optional: use RDS or managed Postgres compatible DB for production Supabase replacement
